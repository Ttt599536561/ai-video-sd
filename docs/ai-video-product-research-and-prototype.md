# AI 生视频网站调研与原型方案

研究日期：2026-06-27

## 1. 调研对象

本次调研参考了即梦 AI、Runway、Luma、Kling、Pika、Vidu、Hailuo AI、OpenAI Sora 等主流 AI 生视频产品。重点观察它们如何组织生成入口、素材上传、模型参数、结果预览、历史视频和付费能力。

## 2. 关键结论

1. 第一版应该直接进入创作台  
主流产品不会把用户先带到复杂营销页，而是把输入、素材、参数和结果集中在一个工作台里。

2. 左侧菜单只保留高频入口  
当前用户端菜单保留：视频生成、项目、购买积分、兑换积分、账号管理。普通用户不显示管理后台入口，管理后台入口仅在服务端确认管理员角色后展示；用户端不再提供独立“任务队列”左侧菜单。素材库、模板、灵感、设置先不放，避免早期产品显得过重。

3. 文生视频和图生视频可以合并  
文字输入是所有模式都会用到的核心能力。用户是否上传图片，可以自然区分文生视频和图生视频，不需要拆成两个 tab。

4. 视频生视频应该单独成 tab  
参考视频是更重的素材类型，上传、校验、成本和处理逻辑都不同，应只在“视频生视频”模式下出现。

5. 参数需要少而清晰  
首版只保留模型、分辨率、比例、时长。风格、运动强度、运镜预设、生成数量先去掉，降低使用门槛。

6. 视频结果区必须清晰表达状态  
生成页右侧预览区应区分空状态、生成中状态和完成可播放状态；完成后自动载入视频并使用浏览器原生播放器。刷新、复制、下载等资产操作不放在生成页预览区，统一放到项目页。

7. URL 和 Key 必须后端保存  
前端只显示模型名、积分消耗、套餐等可公开数据。模型请求 URL、供应商 Key、鉴权策略应只在后台服务或密钥管理系统中保存。

8. 视频和记录只保留 3 天  
用户项目页和管理员记录只显示 3 天内的视频。视频生成页不再提供“最近生成”模块；服务端定时任务应自动清理过期视频文件和记录。

## 3. 当前原型文件

- 前台用户网站：`index.html`
- 用户登录注册页：`auth.html`
- 管理员后台网站：`admin.html`

这些前端文件仍是静态 HTML/CSS/JS，但已接入本地后端 API；登录注册、JWT、积分、兑换码、主要管理能力、PostgreSQL/Prisma 持久化、数据驱动闭环、本地 Mock 视频任务流转、真实供应商代码路径、Redis/BullMQ 状态同步、本地视频存储、参考素材公网 URL 转换和 3 天清理已经具备。真实支付和对象存储仍在后续阶段。

## 4. 前台设计

### 左侧菜单

保留：

- 视频生成
- 项目
- 购买积分
- 管理后台（仅管理员可见）
- 兑换积分
- 账号管理
- 固定在底部的“可用积分”卡片，进度条跟随当前可用积分同步，并展示兑换后积分有效期倒计时

去掉：

- 素材库
- 模板
- 灵感
- 设置

### 视频生成页

生成模式：

- 文/图生视频
- 视频生视频

文/图生视频模式显示：

- 参考图或首帧上传，最多 4 张
- 提示词输入

视频生视频模式显示：

- 参考视频上传，最多 3 个
- 提示词输入

上传规则：

- 文/图生视频模式下，点击“参考图或首帧”区域调用系统文件夹上传图片。
- 视频生视频模式下，点击“参考视频”区域调用系统文件夹上传视频。
- 视频生视频模式外不展示参考视频上传入口。
- 参考音频为可选上传入口，最多 1 个音频文件，随生成任务提交 `audios`。
- 参考图片上传后显示已选图片缩略图列表，避免用户不知道上传了哪几张。
- 上传区域悬浮时展示手型指针、边框高亮和轻微浮起反馈，避免用户误以为不能点击。

生成参数：

- 模型：`video-ds-2.0`、`video-ds-2.0-fast`
- 分辨率：480P、720P、1080P，下拉选择
- 画面比例
- 视频时长：最少 5 秒，最长 15 秒，下拉选择

生成按钮：

- 按钮旁展示“本次消耗 80 积分”
- 去掉“预计等待 1-2 分钟，失败不扣积分”

视频预览区：

- 无活跃任务且无可预览结果时显示空状态。
- 有活跃任务时显示“视频生成中...”。
- 轮询检测到任务完成后自动显示最新完成视频，播放控制使用浏览器原生 `<video controls>`。
- 已移除“最新已下载视频”按钮、播放器说明、刷新/复制/下载工具按钮、写死视频标题和 3 天提示背景区域。
- 全局系统提示改为页面正中间提示；视频生成完成后提示“视频已生成”。
- 当前已完成真实供应商只读模型列表连通、真实提交/状态同步/内容下载代码路径和本地 Mock 成功资产记录；历史一次真实提交返回供应商 `insufficient_user_quota`，用户已说明额度已充值，并要求后续点击生成视频持续真实发起请求。后续用户反馈真实生成已经能成功生成视频。项目页下载和播放均已走资产签名地址。上传参考图/视频/音频时，后端会先保存 reference asset，再向供应商提交公网 URL，公网 API 地址可在后台系统设置配置。

任务队列区域：

- 视频生成页保留任务队列区域，用于展示当前生成中的任务。
- 当前没有任务进行时不显示历史任务，展示“暂无视频队列”。
- 任务队列最多展示 10 条最近任务，超出后在固定高度内滚动；队列卡片底边与左侧“生成参数”卡片底边对齐。
- 视频生成页已移除“最近生成”功能，完成视频统一进入项目页。
- 任务队列只展示任务状态，不提供删除入口；生成任务记录不可由用户删除。

### 项目页

项目页展示用户以往生成完成的所有未过期视频。

排序规则：

- 按生成时间倒序
- 最新生成的视频在最上面最左边

每个视频显示：

- 视频首帧封面
- 标题
- 生成时间
- 模型
- 分辨率
- 时长

每个视频支持：

- 播放；点击后在项目卡片内联播放器直接播放
- 下载；点击后直接下载文件，不进入播放页
- 复制
- 删除；删除项目视频只删除视频资产数据库记录，不影响生成任务记录

### 生成记录页

用户端新增独立“生成记录”页，展示视频生成基础流水：

- 生成时间
- 模型展示名和供应商模型 ID
- 提示词
- 分辨率、尺寸/画幅、用户选择的视频时长
- 参考图片/视频/音频数量
- 扣除积分
- 生成状态
- 视频生成时长，按完成时间减去提交任务时间计算，单位为秒

该页不展示完成时间、视频地址、下载链接、存储 key 或供应商 task id。

### 购买积分页

套餐由后台配置，前端只读取已启用套餐。

原型示例：

- 周套餐：19.9 元，100 积分，有效期 7 天
- 月套餐：69.9 元，500 积分，有效期 30 天
- 团队包：199 元，1800 积分，有效期 90 天

后台可配置项：

- 套餐名称
- 价格
- 积分数量
- 有效期
- 是否启用
- 展示排序

用户端购买积分页不展示“后续后台可配置套餐名称、价格、积分数量、有效期、是否启用和排序；前端不硬编码真实支付规则。”这类实现说明文案。

### 登录注册页

用户端新增 `auth.html`，支持邮箱登录和邮箱注册。

规则：

- 仅支持邮箱。
- 注册不需要邮箱验证码。
- 登录表单包含邮箱和密码。
- 注册表单包含邮箱、设置密码、确认密码。
- 登录、注册、切换 tab 都有静态点击反馈。

## 5. 管理后台设计

管理员后台用于配置前端请求参数、模型、套餐、兑换码、用户、生成记录、操作审计和系统设置。

后台左侧菜单：

- 模型配置
- 积分套餐
- 用户管理
- 生成记录
- 兑换码管理
- 操作审计
- 系统设置
- 返回前台

### 模型配置

支持配置：

- 请求 URL
- 模型名称
- 模型别名
- Key
- 单次消耗积分
- 启用状态，下拉选择“启用 / 禁用”

安全规则：

- URL 和 Key 不能存在前端
- 前端只请求后端暴露的模型列表和积分消耗
- 后端负责调用真实供应商接口

### 积分套餐

支持配置：

- 套餐名称
- 价格
- 积分数量
- 有效期
- 是否启用，下拉选择“启用 / 禁用”
- 排序

后台提供“积分套餐操作台”，管理员可以编辑套餐名称、价格、有效期、积分数、排序和启用状态。

后台交互：

- 保存、新增、编辑、删除、改密、改积分、封禁、解封、播放、下载、复制等操作使用居中弹窗提示。
- 切换模型配置、积分套餐、用户管理、生成记录、操作审计等菜单不弹提示，避免频繁打断。
- 主要管理操作已接真实接口刷新、结构化错误提示和操作审计记录。

### 系统设置

支持配置：

- 公网 API 地址：供应商抓取参考图片/视频/音频时访问的后端公网 origin，例如 `https://api.example.com`。

该地址优先于 `PUBLIC_API_BASE_URL` 环境变量。生产环境必须保证公网可访问且 HTTPS 证书有效，否则供应商抓取参考素材会失败。

### 用户管理

支持：

- 查看用户邮箱
- 查看注册时间
- 查看现有积分
- 查看购买套餐
- 查看用户状态
- 管理用户密码
- 封禁用户
- 解封用户
- 修改用户积分

### 生成视频记录

管理员可以查看所有用户最近 3 天的生成记录。

支持：

- 查看用户
- 查看模型展示名和供应商模型 ID
- 查看提示词
- 查看生成时间
- 查看分辨率、尺寸/画幅、用户选择的视频时长
- 查看参考图片/视频/音频数量
- 查看视频生成时长
- 查看状态

超过 3 天的记录和视频文件自动删除，管理员后台不展示过期数据。
