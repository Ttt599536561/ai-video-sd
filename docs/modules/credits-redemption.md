# 积分、套餐与兑换码

## 积分规则

- 用户生成视频前需要有足够积分。
- 生成按钮旁显示本次消耗积分。
- 用户端左下角显示“可用积分”，并用进度条同步当前可用积分；0 积分时进度条为 0%。
- 用户端左下角同时显示积分有效期倒计时：读取用户兑换记录中最近到期且未过期的兑换记录，显示“还剩 N 天”；永久有效显示“永久”，没有可用兑换记录显示 `--`。

## 套餐规则

- 管理后台可配置套餐名称、价格、有效期、积分数、状态和购买跳转 URL。
- 管理后台套餐操作台默认空；填写后点击“保存套餐”即新增套餐；点击已有套餐“编辑”会填充到操作台，保存后立即更新并清空。
- 套餐删除和启用/禁用调用后端真实 API，失败时会显示明确错误。
- 用户端购买积分页以卡片显示套餐。
- 用户点击套餐购买按钮时，如果该套餐配置了购买 URL，会跳转到对应 URL；未配置时显示未配置提示。
- 购买 URL 只允许 `http://` 或 `https://`，避免危险协议进入前端跳转。

## 兑换码规则

- 管理员批量生成兑换码，默认每个兑换码为 18 位大小写字母+数字随机组合。
- 管理员可选择永久有效或自定义有效天数，不需要选择具体日期时间。
- 管理后台兑换码生成输入框默认空，生成成功后清空。
- 管理后台兑换码页采用上下布局：批量生成兑换码在上方，全部兑换码记录在下方；全部兑换码记录固定高度并通过分页切换，不使用内部滚动条堆长列表。
- 管理后台全部兑换码记录包含完整兑换码、批次、积分、状态、生成时间、兑换信息和“有效期”字段；有效期表示兑换码被兑换后的有效时长，单位为天，永久有效显示“永久”。
- “复制全部兑换码”不放在批量生成表单区；批量生成成功后弹出“本次生成兑换码”弹窗，弹窗内提供复制全部按钮，复制格式为一行一个完整码。
- 管理端历史接口为 `GET /api/admin/redemption-codes`；接口不返回 `codeHash`。
- 完整兑换码通过现有 Prisma 字段 `redemption_codes.code_ciphertext` 持久化，重载时映射回领域对象 `plainCode`；如果老数据只保存过哈希，则无法反推完整码。
- 用户在兑换积分入口输入兑换码。
- 兑换积分页默认不展示用户邮箱；兑换码输入框使用 `name="redemptionCode"` 和 `autocomplete="one-time-code"`，避免浏览器把登录邮箱自动填入兑换码框。
- 后端校验兑换码哈希、有效期、状态和是否已使用。
- 校验通过后给用户增加积分，并记录使用人和使用时间。
- 用户端兑换成功、失败、已兑换、过期、作废和不存在等提示在页面正中间显示；后端结构化错误码会被前端统一映射为中文提示。
- 用户端兑换积分页展示当前用户的脱敏兑换记录：兑换时间、到账积分、有效期、批次名和兑换码前后缀；不展示完整兑换码、哈希或积分流水内部字段。有效期单位为天，永久有效显示“永久”。
