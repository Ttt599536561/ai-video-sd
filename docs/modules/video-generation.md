# 视频生成

## 用户模式

- 文/图生视频：提示词 + 可选参考图或首帧。
- 视频生视频：提示词 + 参考视频。
- 参考素材：图片最多 4 张、视频最多 3 个、音频最多 1 个；图片上传后在上传区展示缩略图列表，便于用户确认素材。前端会把参考图片长边压到 1280px 并转为 JPEG data URL 后再提交，降低供应商处理失败概率。
- 文/图生视频页中参考图/首帧功能与参考音频功能放在同一行；视频生视频页中参考视频功能与参考音频功能放在同一行。
- 提示词下方不再保留“电影感、真实人像、产品展示、竖屏短视频”等快捷风格按钮；生成参数区不再展示“接口参数由后台统一配置”文案。
- 分辨率：480P、720P、1080P。
- 模型：由管理后台配置，用户端通过 `/api/models` 读取已启用模型；用户端只显示后台可编辑的 `displayName`，提交生成时仍发送供应商模型 ID `modelName`。
- 管理员删除模型后，模型会从 `/api/models` 消失；若历史任务引用该模型，后端会软删除配置并保留历史生成记录。
- 时长：5 到 15 秒。
- 生成参数下拉框统一使用 `.select-control` 自定义样式，覆盖模型、分辨率、画幅和时长，避免直接暴露浏览器默认下拉框样式。

## 生成结果

- 点击“生成视频”并通过基础校验后，会立即弹出“正在提交生成任务，请稍候...”，避免真实供应商提交阶段没有反馈；后端创建任务成功后再提示“已提交生成任务”。
- 右侧预览区不再默认展示占位视频：无活跃任务且无可预览结果时显示空状态，有生成中任务时显示“视频生成中...”，任务完成后自动加载最新完成视频到原生 `<video controls>` 播放器。
- 生成完成且已经渲染可播放视频后，前端停止重复轮询；只有存在排队/生成中任务时才继续 1.5 秒轮询，避免已成功视频被不断刷新打断播放。
- 视频生成页已移除预览区左上角“最新已下载视频”按钮、播放器说明、刷新/复制/下载工具按钮、写死视频标题和 3 天提示背景区域；下载、复制、删除等项目资产操作放在项目页处理。
- 用户端左侧菜单不再提供“任务队列”入口。
- 视频生成页仍保留任务队列区域；当前没有生成任务时显示“暂无视频队列”。
- 任务队列最多展示 10 条最近任务，超出后在固定高度内滚动；队列卡片底边与左侧“生成参数”卡片底边对齐。
- 视频生成页已移除“最近生成”模块；历史完成视频统一放在项目页。
- 项目页从 `/api/video/assets` 展示用户已生成成功的视频资产，按生成时间倒序排列；无视频资产时显示无视频空状态，不再展示默认占位封面；有视频资产时会主动加载视频并 seek 首帧作为默认封面，避免切换到项目页时显示黑色视频块；播放按钮在项目卡片内联播放器直接播放，不再弹出提示；下载按钮直接触发文件下载；删除项目视频只删除视频资产记录，不删除生成任务记录。
- 任务队列只展示生成任务状态，不提供删除入口；生成任务记录不可由用户删除。
- 失败任务会在队列卡片中展示 `errorMessage` 的可读摘要，便于区分素材过大、供应商失败、额度不足等原因。
- 用户端已有独立“生成记录”页，通过 `/api/video/job-records` 展示生成时间、模型展示名、供应商模型 ID、提示词、分辨率、画幅/尺寸、用户选择的视频时长、参考图片/视频/音频数量、扣除积分、状态和“视频生成时长”；不再展示完成时间，也不展示视频地址、下载链接、存储 key 或供应商 task id。

## 后端流转

```text
用户提交参数
  -> 后端校验登录、状态、积分、模型
  -> 创建生成任务并扣减积分
  -> VIDEO_PROVIDER_REAL_JOBS=true 时提交真实供应商，否则走本地 Mock Provider
  -> 真实供应商任务由 Redis/BullMQ 后台同步状态并下载内容
  -> 成功写入 OUTPUT_VIDEO 资产，失败退还积分
  -> 用户端轮询刷新任务状态
```

真实供应商任务在 `VIDEO_PROVIDER_REAL_JOBS=true` 且配置 `REDIS_URL` 时由后端 Redis/BullMQ 后台同步器轮询状态；用户端轮询只刷新任务列表，不再批量触发供应商状态查询。当前本地 `backend/.env` 已启用 `VIDEO_PROVIDER_REAL_JOBS=true`，点击生成会真实提交 `POST /v1/videos`。

## 真实供应商接入进展

- 已新增 OpenAI Video 兼容供应商适配器，供应商基础地址配置为后端环境变量 `VIDEO_PROVIDER_BASE_URL`。
- 供应商 API Key 只放在后端环境变量 `VIDEO_PROVIDER_API_KEY`，不放前端。
- 已通过只读接口 `GET /v1/models` 验证供应商连通和鉴权，返回视频模型 `video-ds-2.0`、`video-ds-2.0-fast`。
- 已确认供应商视频协议为 OpenAI Video 风格：提交 `POST /v1/videos`，查询 `GET /v1/videos/{id}`，下载 `GET /v1/videos/{id}/content`。
- 适配器 `OpenAiVideoProvider` 已有 mocked fetch 测试覆盖模型列表、视频提交 payload、状态映射和供应商错误体解析。
- 用户任务流已接入真实供应商提交和状态同步代码路径：`VIDEO_PROVIDER_REAL_JOBS=true` 时创建任务会提交 `POST /v1/videos`，后台同步器会查询 `GET /v1/videos/{id}`；关闭该开关时仍可走本地 Mock。
- 成功状态会下载 `GET /v1/videos/{id}/content` 并写入后端本地存储，生成 `OUTPUT_VIDEO` 资产；失败状态会退款且退款流水幂等。
- 在明确批准下已执行一次真实 `POST /v1/videos` 小流量联调，供应商返回 `HTTP 403: insufficient_user_quota - 用户额度不足, 剩余额度: ＄35.000000, 最低保留额度: ＄9.000000`；未产生 provider task id 或视频资产。
- 用户已说明额度已充值，并要求后续持续允许真实生成；本地真实开关已打开，后续用户反馈真实生成已经能成功生成视频。
- 后端新增管理员测试入口 `POST /api/admin/model-configs/:id/test-provider`，只列出视频模型用于连通检查，不创建供应商视频任务。
- 当前前台模型选择展示后台模型别名，后端生成任务仍按供应商模型 ID 匹配，避免把可编辑展示名传给供应商。
- 参考素材已接入真实提交 payload：图片传 `images`、视频传 `videos`、音频传 `audios`，供应商 Key 仍只在后端环境变量中使用。前端提交 data URL 时，后端会把图片/视频/音频保存到 `VIDEO_STORAGE_DIR` 下的 `references/<jobId>/`，再提交供应商可公网访问的 `/api/video/reference-assets/<jobId>/<filename>` URL；已有 HTTP(S) URL 会原样透传。后端会拒绝超大的图片 data URL，避免明显不适合提交的素材继续触发供应商失败。
- 参考素材公网 URL 的 origin 优先读取管理后台“系统设置”里的公网 API 地址，其次读取 `PUBLIC_API_BASE_URL` 环境变量，再尝试从非本地请求域名推断。该地址必须能被供应商访问，且 HTTPS 证书有效；证书过期会被映射为 `PUBLIC_API_BASE_URL_CERT_INVALID`。
- 部署到新服务器后，本地数据库里的模型配置、积分套餐和系统设置不会自动带过去；需要在新服务器管理后台重新填写公网 API 地址、供应商 URL/Key、模型 ID、积分消耗和积分套餐，除非做数据库备份恢复。
- 如果项目还没上线公网 HTTPS，上传图片生成真实供应商任务会失败，因为供应商无法访问本地电脑生成的 `/api/video/reference-assets/...` URL。`/health` 正常只说明后端健康，不等于本次任务素材文件可被公网抓取。
