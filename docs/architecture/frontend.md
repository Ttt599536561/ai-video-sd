# 前端架构

## 当前实现

- `index.html`：用户主应用，包含视频生成、项目、生成记录、购买积分、兑换积分。
- `auth.html`：邮箱注册登录。
- `admin.html`：管理后台。
- 当前为静态 HTML/CSS/JS 原型，直接调用后端 API。

## API 约定

- 默认 API：本地 `127.0.0.1`/`localhost` 页面使用 `http://127.0.0.1:4000`；公网生产页面默认使用 `window.location.origin`，由 Nginx 将 `/api/` 反向代理到后端。
- 可通过 `localStorage.apiBase` 覆盖。
- 登录成功后保存 `authToken` 和 `authUser`。
- 前端只传生成参数、文件和用户操作，不保存模型 Key 或供应商 URL。
- 用户端管理后台入口不信任本地缓存角色，只有 `/api/me` 实时确认管理员角色后才展示。
- `admin.html` 默认隐藏后台界面，确认管理员角色后添加 `admin-ready` 状态并显示内容。
- 用户端模型、套餐、视频任务队列和项目页通过后端 API 渲染；任务队列读取 `/api/video/jobs`，项目视频读取 `/api/video/assets`。
- 用户端模型下拉只展示管理后台可编辑的 `displayName`，创建任务时提交真实供应商模型 ID `modelName`。
- 用户端参考素材上传限制：图片最多 4 张、视频最多 3 个、音频最多 1 个；图片上传后显示缩略图列表，并在提交前压缩成长边 1280px 的 JPEG data URL；创建任务时分别提交 `images`、`videos`、`audios`。
- 文生视频参考图/首帧与参考音频放在同一行；视频生视频参考视频与参考音频放在同一行；提示词下方快捷风格按钮和“接口参数由后台统一配置”文案已移除。
- 管理端模型、套餐、用户和视频记录列表通过后端 API 渲染，基础操作调用真实接口。
- 管理端模型、套餐和兑换码操作台默认空；保存即新增或更新，保存后清空，编辑时再自动填充。
- 管理端模型配置里的“模型名称”从 `/api/admin/provider-models` 加载供应商只读模型列表，“模型别名”由管理员编辑后给用户展示。
- 管理端兑换码页通过 `/api/admin/redemption-codes` 渲染所有历史兑换码记录；页面采用上下布局，批量生成在上方，全部兑换码记录在下方，记录区固定高度并分页切换；批量生成成功后用弹窗展示本次生成兑换码，复制全部只在弹窗里提供，一行一个。
- 管理端系统设置页通过 `/api/admin/system-settings` 读取和保存公网 API 地址；该地址用于供应商抓取参考素材。
- 管理端系统提示使用居中弹窗；切换模型配置、积分套餐、用户管理、生成记录、操作审计等页签不弹提示。
- 管理端用户列表由后端过滤管理员账号。
- 管理端模型/套餐删除和启用/禁用失败时显示明确错误。
- 管理端“操作审计”页签通过 `/api/admin/audit-logs` 展示最近管理写操作，审计摘要由后端脱敏生成。
- 用户端每 1.5 秒轮询刷新视频状态和生成记录；真实供应商任务状态由后端后台同步器更新，用户端不再自动批量调用同步接口。Mock 回退任务仍可调用本地 Mock 处理路由。
- 用户端检测到生成成功并渲染可播放视频后停止重复轮询；只有仍存在排队/生成中任务时才保持轮询，避免播放器被持续刷新。
- 失败任务队列卡片显示后端保存的 `errorMessage` 摘要，帮助用户理解供应商失败、素材过大或额度问题。
- 视频生成页右侧预览区区分空状态、生成中状态和完成可播放状态；任务完成后自动加载最新完成视频，并通过居中全局提示提醒“视频已生成”。
- 视频生成页任务队列最多展示 10 条，超出后在固定高度内滚动；队列卡片底边与“生成参数”卡片底边对齐。
- 用户端“生成记录”页通过 `/api/video/job-records` 渲染轻量流水，展示生成时间、模型展示名、供应商模型 ID、提示词、分辨率、画幅/尺寸、用户选择的视频时长、参考图片/视频/音频数量、扣除积分、状态和视频生成时长；不展示完成时间，也不读取视频地址、存储 key 或供应商 task id。
- 用户端下载按钮会请求后端 job/asset 级签名下载地址，不直接接触存储目录或供应商内容地址。
- 项目页“播放”按钮会请求视频资产签名地址，并把地址加载到项目卡片内联 `<video>` 播放器；不会把存储目录或供应商内容地址写入前端。
- 项目页“播放”按钮不再弹出提示，只在卡片内联播放器加载视频。
- 项目页“下载”按钮会用签名地址获取 blob 后直接触发文件下载，不进入播放。
- 项目页没有默认占位视频封面；无视频资产时显示无视频空状态，有视频资产时只渲染真实资产卡片，并主动加载视频首帧作为卡片封面，避免初始黑色视频块。
- 任务队列没有删除入口；生成任务记录不可由用户删除。项目页视频卡片的删除按钮调用 `DELETE /api/video/assets/:id`，只删除项目视频资产记录，不影响任务记录。
- 兑换码输入框使用兑换码专用 `name` 和 `autocomplete="one-time-code"`，避免浏览器邮箱自动填充。
- 侧边栏可用积分进度条通过 `updateCreditMeter()` 跟随当前积分同步。
- 侧边栏可用积分有效期通过 `updateCreditValidity()` 从兑换记录计算倒计时，显示“还剩 N 天”、`永久` 或 `--`；该函数必须能在兑换记录尚未加载时安全执行。
- 用户端初始化顺序要求：脚本启动时必须先声明 `redemptionRecords` 等页面状态变量，再调用 `updateUserUI(readStoredUser())` 回填本地用户信息；`updateCreditValidity()` 默认参数不得引用尚未初始化的 `redemptionRecords`，否则刷新页面会触发脚本中断，导致历史视频不渲染、按钮无法点击。
- 生成参数下拉框统一使用 `.select-control` 自定义样式。
- 公共 `apiFetch` 只有请求存在 `body` 时才自动设置 `Content-Type: application/json`；GET/DELETE 等空 body 请求不发送 JSON content-type。
- `index.html`、`auth.html` 和 `admin.html` 均通过 `apiErrorMessage()` 统一处理后端结构化错误码，优先展示后端中文 `message`，再按 `code` 映射登录过期、封禁、余额不足、兑换码、模型/套餐/用户和供应商相关错误。
