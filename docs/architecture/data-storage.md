# 数据与存储

## 推荐选型

- 主数据库：PostgreSQL，当前本地已通过 Docker 启用。
- ORM：Prisma，当前后端已通过 `DATABASE_URL` 默认使用 Prisma 持久化。
- 队列：Redis + BullMQ，当前 Redis 容器已启动；真实供应商任务状态同步已接入后台 BullMQ 扫描任务。用户任务由 `VIDEO_PROVIDER_REAL_JOBS` 控制，当前本地启用真实供应商路径；关闭该开关时可回到 Mock Provider 和 HTTP 处理路由。
- 视频文件：MVP 先放服务器本地目录，默认可通过 `VIDEO_STORAGE_DIR` 覆盖；正式生产建议使用 S3 兼容对象存储。真实供应商成功后会下载内容并写入本地目录，Mock 成功任务仍只写入资产记录。

## 核心数据

- 用户：邮箱、密码哈希、角色、状态、积分、注册时间。
- 模型配置：供应商模型 ID `model_name`、后台可编辑展示名 `display_name`、请求 URL、加密后的 Key、加密 `key_version`、状态、软删除时间 `deleted_at`。
- 套餐：名称、价格、有效期、积分数、状态。
- 兑换码：批次、码哈希、完整码字段、积分数、可为空的过期时间、使用状态。过期时间为空表示永久有效；接口层会把兑换后的有效时长计算为 `validityDays`，单位为天。当前沿用 Prisma 字段 `code_ciphertext` 保存完整码，便于管理员历史复制发给用户。
- 视频记录：用户、生成参数、状态、供应商任务 id、错误信息、创建和完成时间、画幅/尺寸、参考图片/视频/音频数量；作为任务队列/生成历史保留，不允许用户删除。用户端轻量“生成记录”页只读取生成时间、模型名、提示词、分辨率、画幅/尺寸、用户选择的视频时长、媒体数量、扣除积分、状态和生成耗时，不展示视频地址、存储 key 或供应商 task id。
- 视频资产：任务、资产类型、存储 key、MIME、大小、过期和删除时间；项目页展示的是当前用户未删除 `OUTPUT_VIDEO` 资产，用户删除项目视频时删除资产记录，不影响对应任务记录。
- 审计日志：管理员、动作、目标类型、目标 id、脱敏 metadata、IP 和创建时间；覆盖管理端关键写操作，不保存完整模型 Key、兑换码明文或兑换码哈希。
- 系统设置：`system_settings` 当前只保存 `global` 行和 `public_api_base_url`，用于供应商抓取参考素材的公网 API origin；通过管理后台“系统设置”维护。

## 当前本地状态

- PostgreSQL 容器：`backend-postgres-1`，端口 `5432`。
- Redis 容器：`backend-redis-1`，端口 `6379`。
- 数据库连接：`postgresql://postgres:postgres@127.0.0.1:5432/ai_video`。
- 已应用迁移：初始表结构迁移、兑换码永久有效迁移、模型配置软删除字段迁移、积分套餐购买 URL、视频任务记录元数据迁移、系统设置表迁移。
- 不要运行 `docker compose down -v`，否则会删除 PostgreSQL 数据卷。

## 保留规则

- 用户生成视频文件保留 3 天。
- 管理员可见的视频记录同样只保留 3 天。
- 清理任务会删除过期文件，并给资产记录设置 `deleted_at`；真实视频记录清理/隐藏策略后续可继续细化。
- 用户主动删除项目视频时删除视频资产数据库记录；任务队列记录仍保留，保证生成任务历史不可删。
- 模型配置删除时，如果已有视频任务引用该模型，则通过 `deleted_at` 软删除并隐藏配置；历史视频记录继续保留。
- 模型配置 Key 轮换时，旧密文按记录的 `key_version` 解密；缺失版本按 `1` 兼容旧数据，迁移脚本可将旧密文重加密到当前版本。
