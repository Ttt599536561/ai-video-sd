# 长期上下文

## 项目目标

搭建 AI 生视频网站和管理员后台。用户端要完成登录、生成、项目管理、购买积分和兑换积分；管理端要完成模型、套餐、兑换码、用户和视频记录管理。

## 稳定决策

- 数据库已采用 PostgreSQL + Prisma；本地开发通过 Docker PostgreSQL 运行。
- 视频生成属于异步任务，推荐 Redis + BullMQ。
- 生产和本地均通过 `VIDEO_PROVIDER_REAL_JOBS=true` 控制真实供应商提交；开启时用户点击生成视频会真实提交供应商 `POST /v1/videos`，未开启时任务仍可由本地 Mock Provider 驱动。
- 模型 URL 和 Key 必须存在后端环境变量或数据库加密字段，不能存在前端。
- 供应商抓取参考素材所需的公网 API 地址由管理后台“系统设置”维护并持久化到 `system_settings`；该设置优先于 `PUBLIC_API_BASE_URL` 环境变量。地址必须公网可访问，生产 HTTPS 证书必须有效。
- 视频生成任务记录和项目视频资产是两类语义：任务队列用于保留生成历史，不允许用户删除；项目页展示可管理的视频资产，用户删除项目视频只删除资产记录，不影响任务记录。
- 管理端操作审计已接入，覆盖模型、套餐、兑换码批量生成、用户积分和封禁/解封；审计 metadata 只保存脱敏摘要，不保存完整模型 Key、兑换码明文或兑换码哈希。
- 后端 API 错误响应已结构化，包含兼容字段 `error`、稳定错误码 `code`、中文展示文案 `message` 和 `statusCode`；前端三页通过 `apiErrorMessage()` 统一展示常见错误。
- 用户视频文件和管理员可见视频记录只保留 3 天。
- 当前已完成单台公网服务器生产部署，后续可按业务规模再拆分数据库、对象存储和队列。
- 管理后台用户列表不显示管理员账号。
- 模型配置删除采用兼容历史记录的语义：无任务引用则物理删除；已有生成记录引用则软删除并从管理端配置列表和前台模型列表隐藏。
- 前端公共 `apiFetch` 只有请求带 body 时才自动设置 JSON content-type，空 body 的 GET/DELETE 不带 `Content-Type: application/json`。
- 管理端模型名称等于真实供应商模型 ID，由只读供应商模型列表加载为输入建议；如果供应商列表读取失败，管理员仍可手动输入真实模型 ID。模型别名 `displayName` 可编辑，用户前台只展示别名，生成请求仍使用供应商模型 ID。
- 模型配置加密主密钥已支持稳定单 Key 和生产版本化 keyring：`MODEL_CONFIG_ENCRYPTION_KEY_BASE64`/`MODEL_CONFIG_ENCRYPTION_KEY_HEX` 兼容旧配置，`MODEL_CONFIG_ENCRYPTION_KEYS` + `MODEL_CONFIG_ENCRYPTION_CURRENT_KEY_VERSION` 用于轮换；模型记录按 `keyVersion` 解密，缺失版本按 1 兼容。
- 管理后台兑换码页以历史记录为主：管理员可查看所有已生成兑换码记录、完整码和有效期；批量生成成功后只在弹窗展示本次生成兑换码，并在弹窗内复制全部，一行一个。页面采用上下布局，批量生成在上方，全部兑换码记录在下方，记录区固定高度并分页切换。
- 用户端左下角可用积分区域显示积分有效期倒计时，来自兑换记录；最近到期且未过期的兑换记录显示“还剩 N 天”，永久有效显示“永久”，无可用兑换记录显示 `--`。
- 用户端前端启动顺序是稳定约束：必须先初始化 `redemptionRecords` 等状态变量，再执行 `updateUserUI(readStoredUser())`；`updateCreditValidity()` 默认参数不能引用尚未初始化的状态，否则刷新会导致脚本中断、按钮无法点击、历史视频不渲染。

## 当前代码事实

- 前端页面：`index.html`、`auth.html`、`admin.html`，站点图标为 `favicon.svg`。
- 后端目录：`backend/`。
- 后端框架：Fastify + TypeScript。
- 测试框架：Vitest。
- 数据模型：`backend/prisma/schema.prisma`。
- 当前仓储：`PrismaBackedStore` 默认通过 `DATABASE_URL` 使用 PostgreSQL/Prisma 持久化；生产未配置 `DATABASE_URL` 会拒绝启动，只有显式 `USE_IN_MEMORY_STORE=true` 才可回退内存仓储。
- 本地任务处理：`MockVideoProvider` + `POST /api/video/jobs/:id/process`，成功写入 `OUTPUT_VIDEO` 资产，失败退款。
- 真实供应商适配器：`OpenAiVideoProvider` 已支持 `GET /v1/models`、`POST /v1/videos`、`GET /v1/videos/{id}` 和 `GET /v1/videos/{id}/content`；非 2xx 响应会解析供应商错误体并输出具体 `code/message`，同时避免泄露 API Key。
- 参考素材提交契约：图片最多 4 张并传 `images`，视频最多 3 个并传 `videos`，音频最多 1 个并传 `audios`；前端图片上传后显示缩略图列表，参考视频+参考音频原始文件总大小限制为 36MB，后端 `/api/video/jobs` 同步校验数量。
- 真实供应商参考素材转换契约：前端提交 data URL 后，后端保存为 `references/<jobId>/` 下的 reference asset，并把 `/api/video/reference-assets/<jobId>/<filename>` 公网 URL 提交给供应商；已有 HTTP(S) URL 原样透传。
- 真实小流量联调事实：在明确批准下曾提交过 1 次真实 `POST /v1/videos`，供应商返回 `HTTP 403: insufficient_user_quota - 用户额度不足, 剩余额度: ＄35.000000, 最低保留额度: ＄9.000000`；未创建 provider task id 或视频资产。之后用户说明额度已充值，并要求后续点击生成视频持续发起真实请求。
- 后续用户反馈真实生成已经能成功生成视频；当前仍需观察真实闭环稳定性、同步、下载、本地资产和失败兜底。
- 视频内容存储：`VideoFileStorage` 将成功视频写入后端本地目录，生成签名下载地址，并按 3 天过期策略清理文件。
- 项目视频接口：`GET /api/video/assets` 返回当前用户未删除视频资产，播放和下载都先获取资产签名地址；`DELETE /api/video/assets/:id` 删除项目视频资产记录；`DELETE /api/video/jobs/:id` 明确返回 405 `VIDEO_JOB_DELETE_NOT_ALLOWED`。
- 用户端轻量生成记录接口：`GET /api/video/job-records` 返回生成时间、模型展示名、供应商模型 ID、提示词、分辨率、画幅/尺寸、用户选择的视频时长、参考图片/视频/音频数量、扣除积分、状态和视频生成时长；不返回完成时间、视频地址、存储 key 或供应商 task id。
- 管理端兑换码历史接口：`GET /api/admin/redemption-codes` 返回所有已生成兑换码记录，包含完整码、批次、积分、状态、生成时间、兑换信息和 `validityDays`，不返回 `codeHash`。
- 兑换码完整码持久化：Prisma 字段 `redemption_codes.code_ciphertext` 当前用于保存完整兑换码，`PrismaBackedStore` 重载时映射回 `plainCode`；只保存过哈希的老数据无法反推出完整码。
- 本地数据库连接：`postgresql://postgres:postgres@127.0.0.1:5432/ai_video`。
- 模型配置表包含 `deleted_at` 字段，用于支持已引用模型的软删除。
- 系统设置表：`system_settings` 当前保存 `global.public_api_base_url`，用于供应商抓取参考素材。新增/部署该表后必须执行 Prisma migration 和 client generate。
